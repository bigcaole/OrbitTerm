# OrbitTerm 优化升级路线图（P0 / P1 / P2）

更新时间：2026-03-28

## P0（稳定性与安全，优先落地）

- [x] 客户端移除本地备份入口（统一以云同步作为客户端备份路径）
- [x] 管理端备份能力专业化（逻辑 JSON + SQL 快照）
- [x] 管理端备份覆盖范围明确化（配置、账号、同步密文、授权、设备、指令等）
- [ ] 同步链路可观测性增强（统一 `traceId`、失败分层码、管理端同步健康面板）
- [ ] 同步幂等与冲突策略强化（重试、回滚、多端并发冲突细化）
- [ ] 2FA 标准化改造（固定码 -> TOTP）

## P1（可运维性与性能）

- [ ] 角色权限细粒度控制（按模块读写权限点）
- [ ] 自动化备份策略（定时任务 + 保留策略 + 备份完整性校验）
- [ ] 前端包体优化（大 chunk 拆分，按需加载）
- [ ] 大日志列表性能优化（虚拟滚动）
- [ ] 发布流水线升级到 Node 24

## P2（体验与产品化）

- [ ] 客户端快捷指令库内置（Ubuntu / Debian / Alpine / Huawei）
- [ ] 连接页指令库默认折叠，可按设备类型过滤
- [ ] 统一无障碍体验（对比度档位、缩放、键盘可达）
- [ ] 管理端运营能力增强（激活码生命周期、到期提醒、续期策略）

## 执行说明

- 每完成一个条目，更新复选框状态并在 release note 中注明。
- P0 未完成前，不进入 P2 的视觉类“大改”。
